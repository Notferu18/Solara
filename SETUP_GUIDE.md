# Solara Cafe Management System - Setup Guide

## Project Structure

```
solara-cafe-project/
├── solara-backend/          # Laravel API (port 8000)
├── solara-frontend/         # React frontend (port 3000)
└── ml/                      # Python ML server (port 5001)
    ├── app.py               # Flask server for predictions
    ├── train_model.py       # Train Random Forest model
    ├── model.pkl            # Trained model (generated by train_model.py)
    ├── scaler.pkl           # Scaler for features (generated by train_model.py)
    └── data/
        └── solara_sales_data.csv  # Training dataset
```

---

## Backend Setup (Laravel)

### Prerequisites
- PHP 8.3+
- MySQL/MariaDB
- Composer

### Installation
```bash
cd solara-backend

# Install dependencies
composer install

# Setup environment
cp .env.example .env
php artisan key:generate

# Database setup
# Update .env with database credentials (default: solara_db)
php artisan migrate --force

# Start the server
php artisan serve
# Server runs on http://localhost:8000
```

### Test Accounts
After migration, use these credentials:
- **Admin**: `admin@solara.com` / `password`
- **Staff**: `staff@solara.com` / `password`
- **Kiosk**: Open `/kiosk` route without login

## Frontend Setup (React)

### Installation
```bash
cd solara-frontend

# Install dependencies
npm install

# Start development server
npm start
# Runs on http://localhost:3000
```

### Important Notes
- Kiosk is a **public route** — no login required
- Admin/Staff login redirects to `/admin/dashboard` or `/staff/pos`
- Customer accounts **cannot** log in via login page
- Kiosk automatically uses a fallback "Kiosk Guest" user account

## ML Setup (Python Flask)

### Prerequisites
- Python 3.8+
- pip or conda

### Step 1: Install Python Dependencies
```bash
cd ml

# Create virtual environment (optional but recommended)
python -m venv venv

# Activate virtual environment
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# Install packages
pip install pandas scikit-learn joblib flask
```

### Step 2: Train the Model
```bash
# From the ml/ directory
python train_model.py

# Output:
# - model.pkl (Random Forest Regressor)
# - scaler.pkl (StandardScaler for feature normalization)
```

**What it does:**
- Loads `ml/data/solara_sales_data.csv`
- Trains a Random Forest Regressor with 100 trees
- Learns relationships between: category_id, price, day_of_week, month → quantity_sold
- Saves trained model and scaler as pickled files

### Step 3: Start ML Server
```bash
# From the ml/ directory
python app.py

# Server runs on http://localhost:5001
```

**API Endpoint:**
```
POST http://localhost:5001/predict
Content-Type: application/json

{
  "category_id": 1,
  "price": 150.00,
  "day_of_week": 3,
  "month": 6
}

Response:
{
  "predicted_quantity": 42.5
}
```

## Troubleshooting

### Dashboard shows "Loading orders..." forever
- Check backend is running on `http://localhost:8000`
- Verify database connection in `.env`
- Check browser console for API errors

### Reports show no data
- Ensure orders exist in the database with `status='completed'`
- Test orders from kiosk (they're created with status='pending')
- Staff needs to mark orders as 'completed' to appear in reports

### Forecast shows "Could not load forecast..."
- ML server is not running on port 5001 (optional — forecast will use fallback)
- Or model.pkl/scaler.pkl files missing (run `train_model.py` first)
- **This is not critical** — forecasts will use historical averages

### Kiosk orders fail with "Failed to place order"
- Check backend is running
- Verify `kiosk@solara.com` user exists (created automatically by backend)
- Check MySQL error logs for foreign key issues

### "ModuleNotFoundError: No module named 'sklearn'"
- Run `pip install scikit-learn` in your Python environment

## Database Schema Key Points

### Orders Table
- `user_id`: Nullable (kiosk orders use fallback user)
- `queue_number`: Generated for kiosk orders (Q-001, Q-002, etc.)
- `status`: pending → preparing → ready → completed
- `total_amount`: Sum of all order items

### Menu Items
- `stock_quantity`: Used for low stock alerts
- `is_available`: Controls visibility in menus
- `category_id`: Links to categories

### Order Items
- Junction table between Orders and Menu Items
- Tracks quantity and unit_price for each item

## ML Model Details

### Algorithm: Random Forest Regressor
- **Type**: Supervised regression (predicts numerical quantities)
- **Trees**: 100 decision trees
- **Features**: 
  - `category_id` (coffee, meals, snacks, etc.)
  - `price` (item cost)
  - `day_of_week` (1=Monday, 7=Sunday)
  - `month` (1=January, 12=December)
- **Target**: `quantity_sold` (historical sales units)

### Performance Metrics (from training)
- **MAE** (Mean Absolute Error): Average prediction error
- **R² Score**: Model goodness-of-fit (0-1, higher is better)

### Updating the Model
To retrain with new sales data:
```bash
# 1. Ensure ml/data/solara_sales_data.csv is updated
# 2. Run training
python train_model.py
# 3. Restart Flask server (it reloads model.pkl)
python app.py
```
